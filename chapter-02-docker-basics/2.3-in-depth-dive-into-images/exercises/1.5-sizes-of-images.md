# Exercise 1.5 - Sizes of images

## Objective

Understand the massive impact that base image selection has on the final size of a Docker image by comparing a standard OS image (Ubuntu) against a minimal OS image (Alpine Linux).

---

## Concepts Practiced

* Base Image Selection
* Alpine Linux vs Ubuntu
* Inspecting Image Sizes
* Minimal Shells (`sh` vs `bash`)

---

## Exercise Summary

This exercise requires pulling two different variations of the same web service application: one packaged using Ubuntu as the base, and the other using Alpine Linux. By listing the local images, the goal is to observe the stark difference in file size. Additionally, the exercise involves running the Alpine variant interactively to verify it still functions correctly, while noting that minimal distributions often lack standard tools like the `bash` shell, providing only `sh` instead.

---

## Hints

* Use the standard command to list your downloaded images to see their sizes side-by-side.
* When attempting to execute an interactive terminal session inside the Alpine container, remember that `bash` might throw an error because it's not installed. Use `sh` as your command instead.

---

```bash
# Commands explored while solving
docker pull
```

---

## Verification

### Expected Result

When listing the images, the Ubuntu-based image should be significantly larger (often hundreds of megabytes) compared to the Alpine-based image (often less than 20 megabytes). Running the Alpine container interactively with `sh` should allow you to tail the log file just like in the previous Ubuntu exercise.

---

## Challenges Faced

* Attempting to jump into the Alpine container with `bash` instead of `sh` and receiving a "not found" error.

---

## Lessons Learned

* **Size Matters:** Using Alpine Linux can drastically reduce image sizes. Smaller images mean faster download times, cheaper storage, and quicker deployment pipelines.
* **Trade-offs:** The trade-off for Alpine's small size is the lack of common utilities. You must be prepared to use simpler alternatives (like `sh`) or explicitly install the debugging tools you need.
