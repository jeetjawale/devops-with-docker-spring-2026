# Exercise 1.6 - Hello Docker Hub

## Objective

Learn how to utilize Docker Hub as a primary resource for finding image documentation, understanding how an image was built, and discovering how to use it correctly.

---

## Concepts Practiced

- Docker Hub Navigation
- Reading Image Documentation
- Inspecting Public Dockerfiles
- Interactive Container Input

---

## Exercise Summary

The task involves running a specific pre-built image that halts execution and waits for the user to input a specific string (a "secret message"). To figure out what this expected input is, you must navigate to Docker Hub, find the repository for the image, read the provided `README` documentation, and potentially inspect the `Dockerfile` used to build it.

---

## Hints

- You don't need to guess the input. The answer is publicly available on the internet.
- Search for the exact image namespace and name on `hub.docker.com`.
- Maintainers usually provide instructions on how their image behaves in the "Overview" tab. If not, looking at the source `Dockerfile` will reveal the internal logic.

---

## Commands Used

```bash
# Commands explored while solving
docker run -it
```

---

## Verification

### Expected Result

After providing the correct secret message (found via Docker Hub) to the interactive prompt of the running container, the application should accept it and exit successfully.

---

## Challenges Faced

- Realizing that the terminal is intentionally hanging because it is waiting for standard input, not because the container crashed.

---

## Lessons Learned

- **Documentation is Key:** Never blindly run an image without reading its documentation on Docker Hub. You need to know what environment variables, ports, or inputs it expects.
- **Transparency:** Official and well-maintained images provide links to their source `Dockerfile`, allowing you to audit exactly what the container does before running it on your machine.
