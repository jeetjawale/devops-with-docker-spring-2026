# Exercise 1.1 - Getting started

## Objective

Learn how to launch multiple containers in the background, manage their execution state, and selectively stop specific instances while monitoring their status.

---

## Concepts Practiced

- Docker Images
- Container Lifecycle Management
- Background Execution (Detached Mode)
- Process Listing

---

## Exercise Summary

This exercise requires spinning up three instances of a long-running application container (such as a web server) so that they run continuously in the background. After starting them, the task is to stop exactly two of the containers, leaving the third one running. Finally, the state of the system needs to be verified by listing all containers.

---

## Hints

- Remember the specific flag needed to run a container in the background so it doesn't block your current terminal session.
- When stopping a container, you can reference it by either its auto-generated name or its unique Container ID.
- Pay close attention to the `STATUS` column when listing all containers to confirm which ones have `Exited` and which one is still `Up`.

---

## Commands Used

```bash
# Commands explored while solving
docker run
docker ps
docker stop
```

---

## Verification

### Expected Result

Running the command to list all containers should display a list containing the three recently created containers. Two of them should show a status indicating they have exited (e.g., `Exited (0)`), and exactly one should show a running status (e.g., `Up X seconds`).

---

## Challenges Faced

- Ensuring the correct flag was used to prevent the terminal from hanging when the web server container started.
- Identifying the correct IDs to stop the target containers without affecting the one meant to stay alive.

---

## Lessons Learned

- Containers can share the exact same underlying image but run as completely independent, isolated instances.
- The `-d` flag is crucial for running background services like web servers or databases.
- It is highly useful that Docker allows targeting containers using just the first few characters of their ID.
