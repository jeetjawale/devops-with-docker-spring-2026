## Objective

Understand how to effectively clean up unused Docker resources, specifically stopped containers and downloaded images, to free up system disk space.

---

## Concepts Practiced

* Resource Management
* Container Cleanup
* Image Removal
* Docker Daemon Hygiene

---

## Exercise Summary

The objective is to stop any remaining running containers and completely clear the Docker environment of all existing containers and images. This exercise highlights the importance of maintaining a clean workspace and demonstrates the dependencies between images and the containers built from them.

---

## Hints

* You generally cannot remove a running container gracefully without stopping it first. 
* Order of operations is critical: Docker will prevent you from removing an image if there are any containers (even stopped ones) that were created from it. Remove containers before removing images.
* While you can remove items individually by ID or name, Docker provides powerful bulk commands (like `prune`) to clear out unused resources globally.

---

## Commands Used

```bash
# Commands explored while solving
docker stop
docker rm
docker rmi
docker image ls
docker ps -a
docker system prune
```

---

## Verification

### Expected Result

After executing the cleanup commands, the Docker daemon should be completely empty. Running the commands to list all containers (`-a`) and all images should yield empty lists (only the table headers will be displayed).

---

## Challenges Faced

* Encountering error messages when attempting to delete an image before its associated stopped container was deleted.

---

## Lessons Learned

* Docker explicitly protects images from deletion if they are referenced by any container, ensuring system integrity.
* Over time, unused containers and dangling images can consume significant disk space, making regular cleanup essential.
* Bulk removal commands or pruning are much more efficient than deleting resources one by one, especially as a project grows.
