# Exercise 1.15 - Homework

## Objective

Demonstrate the ability to containerize an independent personal project, document its usage, and successfully publish the resulting image to a public Docker Hub repository.

---

## Concepts Practiced

* Custom Image Creation
* Docker Hub Authentication (`docker login`)
* Image Tagging and Namespace Management (`docker tag`)
* Image Publishing (`docker push`)
* Repository Documentation

---

## Exercise Summary

This is a free-form exercise. The task requires selecting any personal project (excluding the course examples), writing a `Dockerfile` for it, building the image locally, and pushing it to a personal Docker Hub account. Additionally, the Docker Hub repository overview page must be updated with basic documentation explaining what the image is and how to run it.

---

## Hints

* **Authentication:** You must run `docker login` in your terminal and provide your Docker Hub credentials before you can push anything.
* **Tagging:** Docker pushes images based on their name. If your local image is named `my-cool-app`, you cannot push it directly. You must tag it to include your Docker Hub username: `docker tag my-cool-app yourusername/my-cool-app`.
* **Documentation:** Go to the web interface of Docker Hub after pushing, navigate to your repository, and edit the "Overview" section to include the `docker run` command necessary to start your app.

---

## Commands Used

```bash
# Commands explored while solving
docker login
docker build -t <username>/<project> .
docker push <username>/<project>
```

---

## Verification

### Expected Result

The image should be publicly accessible on Docker Hub under `https://hub.docker.com/r/<username>/<project>`. The overview tab should contain a readable description and clear execution instructions. Anyone on the internet should be able to run `docker run <username>/<project>` to start the application.

---

## Challenges Faced

* Remembering to correctly format the image tag with the namespace prefix before attempting the push command.

---

## Lessons Learned

* **The Distribution Mechanism:** Docker Hub is the central nervous system of the container ecosystem. Publishing images allows you to easily share software without worrying about the end-user's operating system or installed dependencies.
* **Documentation is Essential:** A published image is useless if the end-user doesn't know which ports to publish or which environment variables are required to run it.
