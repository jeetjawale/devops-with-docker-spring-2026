## Core Concepts Reviewed

### The "Dockerfile" Workflow

Containerizing an application requires understanding the underlying technologies. We learned that the process generally involves reading a project's documentation, selecting an appropriate base image (either a bare OS like Ubuntu or a pre-built language runtime), and codifying the installation and execution steps using instructions like `FROM`, `RUN`, `COPY`, and `CMD`.

### Immutability and Portability

By moving the configuration into a `Dockerfile`, we eliminated dependency and versioning issues. Anyone can pull the resulting image and run the application exactly as intended, regardless of their local operating system.

### The Problem of Scale

While Docker makes running a single application incredibly easy, managing the startup, networking, and teardown of _multiple_ interconnected containers (like the frontend and backend from the later exercises) via raw CLI commands is tedious and error-prone. This sets the stage for Docker Compose in Chapter 3.

## Exercise Retrospective (1.1 - 1.16)

Throughout this chapter, we completed 16 practical exercises that reinforced these concepts:

- **1.1 Getting started & 1.2 Cleanup**: Mastered the basic lifecycle commands (`docker run`, `ps`, `rm`, `rmi`).
- **1.3 Secret message & 1.4 Missing dependencies**: Explored the interactive terminal (`-it`) and demonstrated how ephemeral containers lose state (e.g., manually installed packages) when stopped.
- **1.5 Sizes of images & 1.6 Hello Docker Hub**: Investigated the registry ecosystem and the importance of minimal base images (like Alpine).
- **1.7 Image for script & 1.8 Two line Dockerfile**: Wrote our first `Dockerfiles`, learning the critical distinction between `RUN` (build-time) and `CMD`/`ENTRYPOINT` (runtime).
- **1.9 Volumes**: Escaped the ephemeral filesystem trap by bind-mounting host files into the container.
- **1.10 Ports open**: Breached container network isolation by publishing internal ports to the host machine.
- **1.11 Spring, 1.12 Hello frontend, & 1.13 Hello backend**: Applied our knowledge to containerize real-world applications across different languages (Java, Node.js, Go), dealing with cross-platform architecture issues along the way.
- **1.14 Environment**: Connected two separate containers by injecting configuration via `ENV` variables, realizing the difference between browser-side and server-side execution contexts.
- **1.15 Homework & 1.16 Cloud deployment**: Proved the ultimate value of Docker by publishing a personal image to Docker Hub and successfully deploying it to a public cloud PaaS.
