## Core Concepts Reviewed

### Automated Deployment (CI/CD)

We transitioned from manual `docker build` commands to automated Continuous Integration / Continuous Deployment (CI/CD) pipelines that trigger whenever code is pushed. This removes human error and ensures deployments are consistent.

### The Principle of Least Privilege

We addressed massive security flaws by enforcing non-root users (`adduser -D appuser`). Preventing a container from running as `root` contains the blast radius of a security breach, making it much harder for attackers to compromise the host system.

### Multi-Stage Image Optimization

We tackled image bloat, shrinking gigabyte-sized Node.js and Go images down to just a few megabytes. By utilizing Alpine Linux, `scratch` images, and multi-stage builds, we achieved minimal footprints that directly translate to faster deployments, lower cloud costs, and a significantly reduced attack surface.
