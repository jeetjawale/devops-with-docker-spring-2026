# Exercise 2.9 - Fixup

## Objective

Reconfigure the frontend application to route its API requests through the newly established Nginx reverse proxy.

---

## Concepts Practiced

- Client-side Environment Variables
- Reverse Proxy Routing

---

## Exercise Summary

After setting up the Nginx reverse proxy in Exercise 2.8, the frontend's buttons stopped working. This happens because the frontend was previously configured (via an environment variable at build time or runtime) to make requests directly to the backend's explicit port. This exercise requires fixing the frontend configuration so that all its API calls are directed to the relative `/api/` path (or the absolute proxy URL), ensuring traffic correctly flows through Nginx.

---

## Hints

- Check the browser's developer console (Network tab) to see where the failing requests are currently trying to go.
- Review the frontend's README or previous Dockerfile configurations to find the environment variable that sets the backend URL (often `REACT_APP_BACKEND_URL` or similar).
- Update this environment variable in your `compose.yaml` so the frontend directs API calls to the proxy address (`http://localhost:8000/api/` or just `/api/` depending on the app's internal logic).

---

## Commands Used

```bash
# Restart the stack to apply environment variable changes
docker compose up -d
```

---

## Verification

### Expected Result

After applying the fixes and restarting the stack, all buttons in the frontend application should work correctly. API requests observed in the browser's Network tab should successfully hit `http://localhost:8000/api/...`.

---

## Challenges Faced

- Tracking down exactly where the old backend URL was hardcoded or configured.
- Understanding the difference between browser-side routing (which happens on the host) and container-to-container routing. The frontend code runs in the user's browser, so it must point to the host's localhost proxy port, not the internal Docker network name.

---

## Lessons Learned

- When introducing a proxy, client applications must be updated to respect the new unified URL structure.
- Environment variables passed to frontend containers (like React apps) are often baked in during the build process or evaluated in the browser, meaning they must resolve from the perspective of the *host machine* (the user's browser), not the internal Docker network.
