# Exercise 2.3 - Project with compose (mandatory exercise)

## Objective

Orchestrate a multi-container application by combining separate frontend and backend configurations into a unified Docker Compose setup.

---

## Concepts Practiced

- Multi-container Orchestration
- Service Definitions
- Docker Compose Networking

---

## Exercise Summary

In previous exercises, the frontend and backend of the application were run using separate, lengthy `docker run` commands. This exercise tasks us with consolidating both components into a single `compose.yaml` file. We define both the frontend and backend as separate services, configure their respective port mappings, and ensure they can run together seamlessly with a single command.

---

## Hints

- A `compose.yaml` file can have multiple definitions under the `services:` block.
- Create a service for the frontend and another for the backend.
- Ensure both services map the correct host ports to their respective container ports.
- Refer back to the `docker run` commands used in Exercise 1.14 to extract the necessary ports and environment variables if needed.

---

## Commands Used

```bash
# Bring up the entire application stack
docker compose up

# Bring down the entire application stack
docker compose down
```

---

## Verification

### Expected Result

Executing `docker compose up` successfully starts both the frontend and backend containers. The frontend is accessible via the browser, and the button simulating an API request to the backend functions correctly, proving that the full stack is operational.

---

## Challenges Faced

- Structuring a multi-service YAML file properly.
- Ensuring that there are no port conflicts on the host machine.
- Translating two distinct contexts (frontend and backend) into one cohesive configuration file.

---

## Lessons Learned

- Docker Compose creates a default bridge network for all services defined in the file, allowing them to communicate with each other using their service names as hostnames.
- Consolidating an application stack into Compose drastically reduces the friction of starting, stopping, and sharing the project environment.
