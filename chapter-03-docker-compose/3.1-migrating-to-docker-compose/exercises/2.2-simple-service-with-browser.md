# Exercise 2.2 - Simple service with browser

## Objective

Learn to override the default command of an image and expose ports to the host machine using Docker Compose.

---

## Concepts Practiced

- Docker Compose Port Mapping
- Command Overriding in Compose
- Interacting with Web Services

---

## Exercise Summary

This exercise builds on the previous one. We use the same `simple-web-service` image, but instead of letting it run its default behavior (writing to a log), we need to override its command to start a web server. We then configure port mapping in the `compose.yaml` file to make this web server accessible via a browser on the host machine.

---

## Hints

- Check the Docker Compose specification to find the key used to override the default command (similar to passing arguments at the end of `docker run`).
- Use the `ports` array to map the container's internal port to a port on localhost.
- Ensure you know which port the web service inside the container listens on so you can map it correctly.

---

## Commands Used

```bash
# Start the compose service
docker compose up

# Test the web service from another terminal
curl http://localhost:<mapped-port>
```

---

## Verification

### Expected Result

After running `docker compose up`, you should be able to navigate to `http://localhost:<mapped-port>` in your web browser and see the response from the simple web service.

---

## Challenges Faced

- Finding the correct YAML syntax to override the `CMD` or `ENTRYPOINT` of the underlying Docker image.
- Distinguishing between exposing a port to other containers and mapping it to the host machine.

---

## Lessons Learned

- The `command` key in Docker Compose is a direct translation of appending a command to the `docker run` statement.
- The `ports` key translates to the `-p` flag, effectively bridging the container's network namespace to the host.
